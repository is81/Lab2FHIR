# Lab2FHIR 部署指南

> Windows Server 2019 · Docker · 离线部署（也支持在线）

---

## 部署流程总览

```
┌─ 联网电脑（你的开发机） ─┐         ┌─ 生产服务器（断网）─┐
│                          │         │                      │
│ ① git clone 项目         │         │ ④ 安装 Docker        │
│ ② docker compose build   │  U盘    │ ⑤ docker load 镜像   │
│ ③ docker save → .tar     │ ────→  │ ⑥ 配置 .env          │
│                          │         │ ⑦ docker compose up  │
└──────────────────────────┘         └──────────────────────┘
```

---

## 第一步：在联网电脑上构建镜像

在你的开发电脑上操作（能访问 GitHub 和 Docker Hub）。

### 1.1 下载项目

```powershell
cd C:\
git clone https://github.com/is81/Lab2FHIR.git
cd C:\Lab2FHIR
```

### 1.2 构建镜像

```powershell
docker compose build
```

大约 5-10 分钟。看到 `Successfully built` 即可。

### 1.3 导出镜像为文件

```powershell
docker save -o C:\lab2fhir-images.tar lab2fhir-backend lab2fhir-frontend
```

会生成一个约 500MB 的 `.tar` 文件。

### 1.4 准备离线安装包

把以下文件拷贝到 U 盘或移动硬盘：

| 文件 | 来源 | 说明 |
|------|------|------|
| `C:\lab2fhir-images.tar` | 刚生成的 | Docker 镜像 |
| `C:\Lab2FHIR\` 整个文件夹 | git clone 的 | 项目代码（含 docker-compose.yml） |
| Docker Desktop 安装包 | [下载](https://docs.docker.com/desktop/release-notes/) | 选 Windows 版本 |

> 提示：项目文件夹可以删掉 `frontend/node_modules` 和 `backend/src/uploads/*.pdf` 以减小体积，这些在生产环境不需要。

---

## 第二步：在生产服务器上安装 Docker

把 U 盘插到服务器上。

### 2.1 安装 Docker Desktop

运行 U 盘里的 Docker Desktop 安装包，一路"下一步"。

**注意**：取消勾选 "Use WSL 2 instead of Hyper-V"（Server 2019 不支持 WSL 2）。

安装完重启。

### 2.2 验证

```powershell
docker --version
```

### 2.3 切换 Linux 容器

右下角系统托盘 → 右键 Docker 图标 → **Switch to Linux containers**。

---

## 第三步：加载镜像

### 3.1 导入镜像

```powershell
docker load -i D:\lab2fhir-images.tar
```

（假设 U 盘是 D 盘，按实际盘符修改）

验证：

```powershell
docker images
```

应该看到 `lab2fhir-backend` 和 `lab2fhir-frontend`。

### 3.2 拷贝项目文件

```powershell
# 从 U 盘拷贝项目到 C 盘
Copy-Item -Recurse D:\Lab2FHIR C:\Lab2FHIR
cd C:\Lab2FHIR
```

---

## 第四步：配置密钥和密码

```powershell
# 生成随机 JWT 密钥
$jwtSecret = -join ((48..57) + (97..102) | Get-Random -Count 64 | ForEach-Object { [char]$_ })
Write-Host "JWT 密钥: $jwtSecret"
```

复制输出的字符串，然后：

```powershell
@"
LAB2FHIR_JWT_SECRET=粘贴刚才生成的字符串
LAB2FHIR_ADMIN_PASSWORD=设置管理员密码
LAB2FHIR_DOCTOR_PASSWORD=设置医生密码
CORS_ORIGINS=*
"@ | Out-File -FilePath .env -Encoding UTF8
```

---

## 第五步：启动

```powershell
docker compose up -d
```

验证：

```powershell
docker compose ps
```

两个都是 `Up` 即可。

### 创建用户账户

```powershell
docker compose exec backend python tools/seed_users.py
```

---

## 第六步：打开防火墙

**PowerShell（管理员）**：

```powershell
New-NetFirewallRule -DisplayName "HTTP 80" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow
```

浏览器打开：**`http://服务器IP`**

---

## 后续：更新版本（离线）

当有新版本时，在联网电脑上：

```powershell
cd C:\Lab2FHIR
git pull
docker compose build
docker save -o C:\lab2fhir-update.tar lab2fhir-backend lab2fhir-frontend
```

把 `.tar` 文件拷到服务器，在服务器上：

```powershell
cd C:\Lab2FHIR
git pull   # 如果有 git（从联网电脑也拷了项目文件就不需要）
docker load -i D:\lab2fhir-update.tar
docker compose up -d
```

## 日常管理

```powershell
docker compose logs -f backend    # 查看后端日志
docker compose restart            # 重启服务
docker compose stop               # 停止
docker compose start              # 启动
```

### 忘记密码

```powershell
docker compose exec backend python -c "
import sys; sys.path.insert(0, 'src')
from src.db.models import SessionLocal
from src.db.user_models import User
from src.auth.security import hash_password
db = SessionLocal()
user = db.query(User).filter(User.username == 'admin').first()
user.password_hash = hash_password('新密码')
db.commit(); db.close()
print('已重置')
"
```

## 故障排查

| 问题 | 解决方法 |
|------|----------|
| `docker` 不识别 | Docker Desktop 是否在运行（右下角图标） |
| 浏览器打不开 | 检查防火墙 `Get-NetFirewallRule -DisplayName "HTTP 80"` |
| 页面空白 | 等 30 秒刷新，首次加载较慢 |
| 镜像导入失败 | 检查 .tar 文件完整性，网络传输可能损坏 |
