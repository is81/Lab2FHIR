# Lab2FHIR 部署指南

> 小白友好版 · Ubuntu/Debian · Docker · IP 访问

---

## 准备工作

### 你需要准备

| 项目 | 说明 |
|------|------|
| 一台 Ubuntu 服务器 | 建议 2核4G 以上（云服务器约 ¥50-100/月） |
| 服务器的 IP 地址 | 如 `123.456.789.0` |
| SSH 客户端 | Windows 用 PowerShell 或 [MobaXterm](https://mobaxterm.mobatek.net/) |

### 连接到服务器

打开 PowerShell（Win+R → 输入 `powershell`），输入：

```bash
ssh root@你的服务器IP
```

输入密码后进入服务器。

---

## 第一步：安装 Docker

逐行复制执行：

```bash
# 1. 更新系统
apt update && apt upgrade -y

# 2. 安装 Docker
curl -fsSL https://get.docker.com | bash

# 3. 启动 Docker
systemctl enable docker && systemctl start docker

# 4. 验证安装
docker --version
```

看到版本号就说明成功了（如 `Docker version 27.x.x`）。

---

## 第二步：下载项目

```bash
# 安装 git（如果还没有）
apt install git -y

# 克隆项目
git clone https://github.com/is81/Lab2FHIR.git
cd Lab2FHIR
```

---

## 第三步：配置密码和密钥

**这是最重要的一步，必须做！**

```bash
# 生成随机 JWT 密钥（复制输出的那串字符）
openssl rand -hex 32
```

记下输出的字符串（如 `a1b2c3d4...`），然后创建配置文件：

```bash
# 创建 .env 文件
cat > .env << 'EOF'
# ====== 必填 ======
LAB2FHIR_JWT_SECRET=把你刚才生成的随机字符串粘贴到这里
LAB2FHIR_ADMIN_PASSWORD=设置一个管理员密码
LAB2FHIR_DOCTOR_PASSWORD=设置一个医生密码

# ====== 选填（以下保持默认即可）======
CORS_ORIGINS=*
EOF
```

> ⚠️ 把 `LAB2FHIR_JWT_SECRET=` 后面的内容替换成你刚才生成的随机字符串。  
> ⚠️ 把密码换成你自己的，不要用示例值。

---

## 第四步：启动服务

```bash
docker compose up -d
```

首次启动会下载镜像和依赖，大约需要 2-5 分钟。看到 `Started` 或两个 `done` 就成功了。

验证：

```bash
docker compose ps
```

应该看到两个服务都是 `Up` 状态：

```
NAME                  STATUS
lab2fhir-backend      Up
lab2fhir-frontend     Up
```

---

## 第五步：初始化数据库和用户

```bash
# 进入后端容器
docker compose exec backend python tools/seed_users.py
```

这一步会创建登录账户。输出类似：

```
已创建默认用户:
  admin  / 你设置的密码  (pathology_staff - 病理科)
  doctor / 你设置的密码  (doctor - 科室医生)
```

---

## 第六步：打开防火墙

云服务器默认可能屏蔽了 80 端口，需要放行：

```bash
# 方法 1：云厂商控制台
# 登录阿里云/腾讯云/AWS 后台 → 安全组 → 入方向 → 添加规则
# 端口：80  来源：0.0.0.0/0

# 方法 2：服务器上执行（如果有 ufw）
ufw allow 80
```

---

## 第七步：访问系统

浏览器打开：**`http://你的服务器IP`**

你会看到：
1. **首页仪表盘** — 统计数据、报告类型分布
2. **导入报告** — 拖入 PDF，点"开始导入"，输入 admin 账户登录
3. **报告列表** — 搜索、筛选已导入的报告

---

## 后续操作

### 导入 PDF 报告

1. 点击侧边栏"导入报告"
2. 拖入 PDF 文件（或点击选择）
3. 点"开始导入"
4. 弹出登录框 → 输入 admin 账户 → 开始导入

### 查看日志

```bash
docker compose logs -f backend    # 后端日志
docker compose logs -f frontend   # 前端日志（nginx）
```

### 停止/重启

```bash
docker compose stop               # 停止
docker compose start              # 启动
docker compose restart            # 重启
docker compose down               # 完全关闭并删除容器
```

### 更新到最新版本

```bash
cd ~/Lab2FHIR
git pull
docker compose up -d --build
```

### 添加新用户

```bash
docker compose exec backend python -c "
import sys; sys.path.insert(0, 'src')
from src.db.models import SessionLocal
from src.db.user_models import User, UserRole
from src.auth.security import hash_password

db = SessionLocal()
db.add(User(
    username='新用户名',
    password_hash=hash_password('新密码'),
    role=UserRole.DOCTOR.value,
    display_name='显示名'
))
db.commit(); db.close()
print('用户已创建')
"
```

## 故障排查

| 问题 | 解决方案 |
|------|----------|
| 浏览器打不开 | 检查防火墙是否放行 80 端口 |
| 页面显示但数据为空 | 登录 admin 账户 → 导入 PDF |
| 导入时提示"网络错误" | `docker compose restart backend` |
| PDF 无法查看 | `docker compose logs backend` 查看错误 |
| 忘记密码 | 执行上方"添加新用户"脚本，或修改已有用户密码 |

## 进阶：配置 HTTPS（可选）

当你有了域名后，可以免费配置 HTTPS：

```bash
# 1. 安装 nginx + certbot
apt install nginx certbot python3-certbot-nginx -y

# 2. 获取证书
certbot --nginx -d 你的域名.com

# 3. certbot 会自动配置续期
```

有问题看日志：`docker compose logs -f`
