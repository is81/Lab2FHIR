📋 项目概述
Lab2FHIR 是一个开源的轻量级微服务，旨在将非结构化的化验单（PDF或图片）自动、准确地转换为标准的HL7 FHIR R4格式。它如同一个“化验单数据翻译官”，打通非结构化数据与现代化医疗信息系统之间的壁垒。

🎯 项目目标
核心目标：提供一个简单、可靠的REST API，接收化验单文件，返回标准的FHIR Bundle（包含DiagnosticReport和Observation资源）。

最终愿景：成为医院IT团队将历史化验单数字化的首选开源工具，为临床数据分析和互操作性奠定基础。

🧰 技术栈选型
后端：Python 3.9+。生态丰富，适合AI/ML任务，且已有成熟的转换库可借鉴。

Web框架：FastAPI。高性能，自动生成API文档，便于开发者使用。

OCR引擎：PaddleOCR 或 Tesseract。用于从图片或扫描版PDF中提取文字。项目初期建议先用pdfplumber等库处理文本型PDF。

AI模型 (可选)：Gemma 或 MedGemma。用于处理格式复杂、规则难以覆盖的化验单，提高提取的泛化能力。

术语映射：自建轻量级LOINC映射知识库。

部署：Docker。提供一致的运行环境，简化部署流程。

开发路线：
基于pdf_test文件夹内样本，核心思路应该是 “文本提取优先，规则解析为主，AI/OCR仅作为备选” 。因为这些报告都是文本型PDF，内容可直接提取，无需OCR。

第一阶段：核心解析引擎（第1-6周）
目标：实现对上述三类报告PDF的文本提取和关键信息解析。

第1-2周：PDF文本提取与清理

工具：使用 pdfplumber 或 PyMuPDF（fitz）提取文本，保留坐标信息。

工作：编写脚本，分别提取三份报告示例中的原始文本，并观察其布局规律。

关键点：注意处理“姓名”、“性别”、“年龄”、“送检科室”、“报告日期”、“标本类型”等通用字段。

第3-4周：针对三种报告类型的专用解析器

DNA倍体解析器：

提取顶部的患者信息和结论文本。

关键挑战：提取底部的表格数据（Percent, DI均值, STD）。可以使用正则或按行坐标定位。

TCT解析器：

提取“细胞项目”和“微生物项目”下的所有“有/无”条目。

关键点：将 鳞状上皮细胞：有 这类文本解析为 { "item": "鳞状上皮细胞", "result": "有" }。

HPV解析器：

提取“高危型HPV”和“低危型HPV”的结论（阳性/阴性）。

如果为阳性，提取具体分型（如“HPV54”）。

第5-6周：信息合并与标准化

设计一个内部数据模型（如 LabReport 类），包含患者信息、报告类型、观察值列表等。

将三个解析器的输出统一到这个模型中。

第二阶段：FHIR映射与输出（第7-10周）
目标：将解析后的数据转化为标准的FHIR Bundle。

第7-8周：FHIR基础映射设计

固定映射：

患者信息（姓名、性别、年龄） → Patient 资源（或直接放入 DiagnosticReport.subject）。

报告本身 → DiagnosticReport 资源。

每一项检测结果 → Observation 资源。

LOINC映射（关键）：

为“DNA倍体”、“TCT”、“HPV”这三类报告建立核心映射表。

例如："高危型HPV" → LOINC 21440-3（HPV DNA检测）。

初期可以手动映射，后续再考虑自动化。

第9-10周：FHIR生成器开发

编写代码，将内部数据模型转换为FHIR R4 JSON格式。

使用 fhirclient 或 fhir.resources 等库辅助生成。

对生成的Bundle进行结构性验证。

第三阶段：API与产品化（第11-16周）
目标：将引擎包装为易用的服务，并进行测试与推广。

第11-12周：REST API开发

使用FastAPI开发 /convert 端点。

支持上传PDF文件，返回JSON格式的FHIR Bundle。

增加简单的错误处理（如“不支持的文件格式”）。

第13-14周：配置与扩展

规则配置化：将解析规则（如正则表达式）抽离为配置文件，方便用户根据自己医院的报告模板进行调整。

批量处理：增加批量上传或处理目录下所有PDF的功能。

第15-16周：测试与文档

用更多真实样本进行测试（注意脱敏）。

编写清晰的README、API文档和示例。

💡 关键技术建议
文本坐标利用：pdfplumber 可以获取每个字符的坐标。对于表格数据，利用坐标定位比纯正则更可靠。例如，DNA报告底部的表格，可以通过 bbox 参数截取特定区域进行解析。

LOINC映射策略：建议采用本地知识库 + 后备规则。先建一个包含常见检查项（如“高危型HPV”、“鳞状上皮细胞”）的本地CSV映射表，如果找不到对应项，再用规则生成一个临时的编码。

代码模块化：将“通用信息提取”、“报告类型识别”、“专用解析器”、“FHIR生成器”等模块分离，方便后期维护和扩展。
